# Overview

For this assignment, I built a full end-to-end lead automation system that simulates how a business would handle incoming client inquiries. The system accepts a lead form submission, validates the input, enriches company data using web scraping, generates a personalized business audit report using the Gemini API, converts it into a PDF, and automatically emails it to the client. It also logs each lead into Google Sheets and uploads a copy to Google Drive for tracking and backup.

The goal was to fully automate the workflow from form submission to final delivery without any manual steps.

---

# How the system works

The process starts when a POST request is sent to `/submit-lead`. The backend validates the input using the `validator` library, ensuring a valid email, required fields like company name, and a properly formatted website URL if provided.

If a website is provided, the system performs multi-level web scraping. First, it scrapes the homepage using Axios and Cheerio, extracting structured data such as the page title, meta description, keywords, headings, and paragraphs.

It then identifies relevant internal links (such as “about”, “services”, “products”, or “solutions”) and scrapes those pages as well to enrich the dataset with deeper contextual information.

All scraped data is cleaned and filtered to remove noise and irrelevant content before being used.

This enriched dataset is then sent to the Gemini API with a structured prompt. The model is instructed to act as a senior business consultant and generate a professional audit report with sections including:

- Executive Summary  
- Company Positioning Analysis  
- Website & Messaging Review  
- Brand & Trust Signals  
- Strengths  
- Weaknesses  
- Opportunities  
- Recommendations  
- Personalized Outreach Email  

The prompt also enforces strict rules to ensure the model only uses real scraped data and avoids hallucinations.

Once generated, the report is converted into a PDF using PDFKit with structured formatting and section headers.

The PDF is then emailed to the client as an attachment using Nodemailer.

Finally, the lead is logged into Google Sheets and the PDF is uploaded to Google Drive for storage and tracking.

To improve efficiency, external operations (email, Drive upload, Sheets logging) are executed in parallel.

---

# Key design decisions

- Single Express server for simplicity and end-to-end focus  
- Gemini API used for strong text generation and easy REST integration  
- Cheerio used instead of Puppeteer for lightweight static scraping  
- Multi-page scraping for richer context beyond just the homepage  
- Strong prompt engineering to prevent hallucinations and enforce structure  
- Fallback response system if AI generation fails  

---

# Tradeoffs and limitations

- Scraping is limited to static HTML and may not work on JS-heavy sites  
- No job queue system, which limits scalability under heavy load  
- PDF output is text-based and not visually designed  
- Output quality depends heavily on Gemini API and scraped data quality  

---

# Error handling and reliability

The system includes error handling for:

- Gemini API failures  
- Website scraping issues  
- Email sending  
- Google Drive upload  
- Google Sheets logging  

If AI generation fails, a fallback report is still returned so the workflow completes.

External services are isolated so failures do not break the entire pipeline.

---

# AI usage

The Gemini API is used as the core intelligence layer. The prompt is carefully structured to:

- Use only real scraped data  
- Avoid hallucinated features  
- Produce structured consulting-style output  
- Clearly distinguish between client and target company
